# 初生版本 V1 需求文档

## 版本定位

初生版本的目标不是功能完整，而是验证这个软件最核心的价值：

> 在 macOS 上做出轻量、低延迟、压感自然、稳定器顺滑的绘画体验。

这个版本要让用户打开软件后能完成一张基础插画：新建画布、画线稿、上色、使用基础图层、创建快照、保存工程、导出图片。

第一版暂不追求高级辅助、人偶、复杂透视、水墨系统、PSD 完整兼容。所有资源都优先投入到笔刷手感、画布性能、图层稳定性和快照系统。

## 核心成功标准

- 笔尖跟手，稳定器不会产生明显拖拽感。
- 线条顺滑但不死板，保留自然手绘走势。
- `6000 x 6000` 以下画布在常规图层数量下绘制顺滑。
- `8000 x 8000` 级别画布在较多图层下仍可稳定绘制。
- 快照节点可以完整返回某一瞬间。
- 软件界面干净，不打扰画画。

## 目标性能

### 画布和图层

- `4000 x 4000` 以下：目标支持 `300-500` 个常规绘画图层。
- `6000 x 6000` 以下：目标支持 `150-300` 个常规绘画图层。
- `8000 x 8000` 以下：目标支持 `50-100` 个常规绘画图层。
- 超过 `8000 x 8000`：允许后续开放，但第一版不作为主要优化目标。

### 绘制体验

- 正在绘制的笔触必须优先渲染。
- 画布拖动、缩放、旋转应保持流畅。
- 图层缩略图、自动保存、快照整理等后台任务不能抢占笔刷输入。

## 平台范围

- 第一优先目标设备：用户当前 MacBook Air，Apple M5，10 核 CPU，24GB 内存，macOS 26.4.1。
- 第一优先数位板路线：Wacom。
- 当前临时测试板：Wacom 影拓 5 / Intuos5。
- 后续重点测试板：Wacom Intuos Pro M / PTH-660。
- 第一版所有手感、压感、画布性能和快照体验，先以这套真实绘画环境为准。
- 优先支持 Apple Silicon Mac。
- 旧系统兼容目标：macOS 10.12 Sierra。
- 如果核心绘画体验需要较新的系统 API，旧系统可以采用降级实现，但不得影响第一优先目标设备上的绘画手感。
- 优先适配主流数位板：Wacom、Huion、XP-Pen。
- 支持鼠标和触控板基础绘画操作。

### 设备优先级

第一版不是以“理论兼容最多设备”为最高目标，而是先做出在用户真实设备上足够好用的绘画工具。

优先级顺序：

1. 用户当前 MacBook Air + Wacom 路线。
2. Wacom Intuos Pro M / PTH-660 实机测试。
3. 用户当前影拓 5 / Intuos5 的尽力兼容。
4. 现代 Apple Silicon Mac + 主流新 Wacom/数位板。
5. Intel Mac 和其他主流数位板。
6. macOS 10.12 旧系统和 Bamboo/旧 Intuos 等旧设备。

### macOS 10.12 兼容目标说明

保留 macOS 10.12 兼容目标的主要原因是照顾旧 Wacom 用户，尤其是 Bamboo 和部分旧 Intuos 设备。很多旧数位板最终官方驱动停留在 macOS 10.12 或接近版本。

但 macOS 10.12 很老，和现代 macOS/Apple Silicon 开发环境差异明显，因此需要分级处理：

- 当前主开发和手感验证以用户的 macOS 26.4.1 设备为准。
- macOS 10.12 作为旧系统兼容目标，优先保证打开、绘制、压感、保存和导出等基础能力。
- 高级 UI、颜色管理、GPU 优化、Time-lapse、高级辅助等功能可根据系统能力降级。
- 如果旧系统支持成本明显损害主线绘画体验，应优先保证用户当前 Mac 和现代 macOS 体验，再评估旧系统独立兼容包。

### Wacom 影拓 5 兼容性注意

Wacom 影拓 5 / Intuos5 是当前临时测试板，同时也是旧型号。官方资料中该系列最终驱动为 Wacom Tablet Driver 6.3.46-2，官方支持范围包含 macOS 10.14-12。当前第一优先设备运行 macOS 26.4.1，因此第一阶段需要验证：

- 当前系统是否能稳定识别影拓 5。
- 应用是否能收到笔压、坐标、按钮、橡皮等 tablet event。
- 读取到的 pressure 是否平滑、连续、无明显跳变。
- Wacom 驱动权限、输入监控、辅助功能权限是否会影响绘画。
- 如果官方驱动链路不稳定，需要提供降级兼容策略或明确提示。

这项验证应排在笔刷手感原型之前。只有输入链路稳定，后续稳定器和笔刷引擎测试才有意义。

当前已完成的初步检查：

- macOS 26.4.1 USB/HID 层可以识别设备为 `Intuos5 touch M`。
- Vendor ID 为 `0x56a`，Product ID 为 `0x27`。
- 浏览器 Pointer Events 测试页只能得到最高 `0.5` 的默认按下压力，说明浏览器层没有拿到真实 Wacom 压感。
- 原生 AppKit/NSEvent 测试器目前只能得到 `0` 和 `1`，线条没有连续粗细变化，说明当前无驱动或当前权限链路下 AppKit 也没有拿到真实 Wacom 压感。
- 底层 HID 原始报告测试器在 Terminal 中使用管理员权限后可以读取报告。
- 当前捕获到的主要是 `id=1 len=5` 的短报告，数据表现更像鼠标兼容/相对输入模式；未看到可直接确认连续压感的完整 tablet report。
- 初步判断：硬件和 USB/HID 通路可用，但当前无 Wacom 驱动初始化时，设备可能没有进入完整压感数位板模式。
- 若后续不再以影拓 5 作为主要输入设备，则该问题不阻塞 1.0 主体开发；后续可在兼容层继续研究 Wacom 官方旧驱动方案，或研究是否能在应用内/辅助组件中初始化 Intuos5 的完整 HID tablet 模式。

### Wacom PTH-660 适配策略

Wacom Intuos Pro M / PTH-660 将作为 1.0 手感校准的重点测试板。该设备属于较新的 Intuos Pro 系列，Wacom 当前驱动页面仍列出 PTH-660，适合作为现代 macOS 上的主要 Wacom 验证对象。

在 PTH-660 接入之前，1.0 仍可继续开发：

- 输入层先抽象为 `PointerInput` / `TabletInput`，不把笔刷引擎绑定到某一块板。
- 鼠标和触控板提供无压感输入。
- 开发模式提供模拟压感曲线，用于调试笔刷、稳定器和渲染。
- Wacom 输入适配层预留 pressure、tilt、rotation、buttons、eraser 等字段。
- 等 PTH-660 接入后，再校准真实采样率、压力曲线、倾斜和侧键。

因此 1.0 的主体开发不需要等待新数位板。真正依赖 PTH-660 的部分是最终手感校准，而不是画布、图层、快照、文件格式和笔刷架构。

### 旧 Wacom 设备兼容策略

许多旧 Wacom 数位板和数位屏都存在类似问题：硬件本身仍可用，但官方驱动已经停止支持较新的 macOS。

需要重点关注的旧型号包括：

- Bamboo / Bamboo Pen / Bamboo Fun / Bamboo Comic：常见型号如 CTL-470、CTH-470、CTH-670、CTL-460、CTH-460、CTH-461、CTH-661、MTE-450、CTE-450、CTE-650。
- Intuos 3：常见型号如 PTZ-430、PTZ-630、PTZ-930、PTZ-1230。
- Intuos 4：常见型号如 PTK-440、PTK-640、PTK-840、PTK-540WL。
- Intuos 5 / Intuos5 Touch：常见型号如 PTK-450、PTK-650、PTH-450、PTH-650、PTH-850。
- 旧 Intuos 2013：常见型号如 CTL-480、CTH-480、CTH-680。
- Intuos 2015：常见型号如 CTL-490、CTL-690、CTH-490、CTH-690。
- 部分旧 Cintiq：如 Cintiq 12WX、21UX、22HD、24HD、13HD、27HD 等，具体支持范围按型号确认。

第一版不要承诺所有旧设备都完整支持。更合理的策略是：

- 应用本身尽量使用 macOS 标准 tablet event 和 pointer event，不强绑定某个 Wacom 私有 SDK。
- 首次启动或设置页提供“输入诊断”工具，检测坐标、压感、倾斜、橡皮、侧键是否可用。
- 如果驱动只提供坐标但没有压感，软件应明确提示“当前驱动未提供压感”。
- 如果旧设备在新 macOS 上依赖第三方修复驱动，软件只保证尽量兼容收到的系统事件，不承诺驱动安装和系统级修复。
- 开发和测试优先级：Wacom 路线最高，其中 PTH-660 作为后续重点测试板；影拓 5 作为旧设备尽力兼容；再其次是仍有新 macOS 官方驱动支持的常见 Wacom/数位板设备和其他品牌设备。

这能避免软件背上“修复所有旧 Wacom 驱动”的负担，同时尽量照顾大量仍在使用旧 Bamboo、旧 Intuos 的用户。

## 核心架构方向

### 渲染

- 使用 macOS 原生技术。
- 画布区域优先使用 AppKit `NSView` + Metal。
- UI 可使用 SwiftUI + AppKit 混合。
- 大画布采用 tile 分块渲染，不使用单张巨大贴图承载全部画布。

### 图层

- 图层使用稀疏 tile 存储。
- 空白区域不占用完整图层内存。
- 图层合成使用 GPU 缓存。
- 修改时只更新受影响 tile。

### 历史和快照

- 普通撤销、手动快照、自动保存三套系统分开。
- 底层可以共享 tile 版本机制，但用户界面和用户心智必须分开。

## 基础功能范围

### 文件

- 新建画布。
- 打开项目文件。
- 保存项目文件。
- 另存为项目文件。
- 导出 PNG。
- 导出 JPEG。

### 项目文件

项目文件需要保存：

- 画布尺寸。
- 色彩空间。
- 图层内容。
- 图层顺序。
- 图层可见性。
- 图层透明度。
- 图层混合模式。
- 快照节点。
- 普通历史必要数据。
- 自动恢复必要数据。
- 笔刷基础设置。

建议使用 macOS package 格式，扩展名后续命名，例如 `.drawdoc`、`.inkdoc` 或其他正式名称。

### 画布操作

- 缩放。
- 平移。
- 旋转。
- 镜像视图。
- 重置视图。
- 显示透明背景棋盘格。
- 设置背景色。

### 图层

第一版需要支持：

- 新建图层。
- 删除图层。
- 复制图层。
- 调整图层顺序。
- 显示/隐藏图层。
- 锁定图层。
- 透明像素锁定。
- 剪贴图层。
- 图层透明度。
- 基础混合模式：
  - 正常
  - 正片叠底
  - 滤色
  - 叠加

图层组可以暂缓，但文件结构要预留。

### 颜色

第一版颜色管理采用轻量但正确的方案：

- 默认工作空间：sRGB IEC61966-2.1。
- 可选工作空间：Display P3。
- 导入图片时读取嵌入 ICC profile，并转换到当前画布工作空间。
- 导出 PNG/JPEG 时嵌入 ICC profile。
- 显示端使用 macOS ColorSync/系统显示 profile。
- 第一版不做真正 CMYK 绘制。
- CMYK 预览和印刷导出后续版本再做。

颜色工具：

- 色轮。
- HSV/RGB 输入。
- 最近使用颜色。
- 基础调色板。
- 吸管工具。

## 笔刷系统

### 笔刷架构

第一版采用位图 stamp 笔刷架构，方便后续扩展自定义笔尖、纸纹、水彩、水墨和笔刷包。

### 第一版笔刷类型

- 线稿笔。
- 铅笔。
- 喷枪。
- 橡皮。
- 基础水彩笔。
- 基础涂抹工具。

### 基础参数

- 大小。
- 最小大小。
- 浓度/不透明度。
- 最小浓度。
- 流量。
- 硬度。
- 间距。
- 笔尖形状。
- 纸纹/笔纹纹理强度。
- 压感控制大小。
- 压感控制浓度。
- 压感曲线。
- 稳定器。
- 自然修线：关闭 / 轻微 / 标准。

### 稳定器目标

稳定器是第一版最重要的功能之一。

目标体验：

- 笔尖仍然跟手。
- 轨迹被聪明地修正。
- 不像 Photoshop 那样有明显拖拽感。
- 不把线条修成机械矢量线。
- 快速长线自动更稳。
- 慢速细节保留手部变化。

界面上不要暴露复杂算法概念，只提供：

- 稳定器强度。
- 自然修线开关。
- 速度响应：自然 / 跟手 / 稳定。
- 压感曲线。

### 基础水彩

第一版做轻量水彩，不做真实流体模拟。

需要支持：

- 混色。
- 水分/稀释。
- 残留/拖色。
- 压感影响浓度和混色程度。

### 基础涂抹

第一版需要提供基础涂抹工具。

参数：

- 大小。
- 强度。
- 硬度。
- 间距。
- 压感控制大小。
- 压感控制强度。

第一版只要求当前图层涂抹。采样所有可见图层后续再做。

## 快照和历史

### 普通撤销

- 默认保留 `500` 步。
- 普通撤销是线性历史，适合连续回退操作。
- 普通撤销不等于快照。

### 手动快照

快照是完整状态节点。

创建快照时保存当时整个文件状态，包括：

- 所有图层内容。
- 图层顺序。
- 图层可见性。
- 图层透明度。
- 混合模式。
- 剪贴关系。
- 透明锁定状态。
- 画布尺寸。
- 色彩空间。
- 必要的工具状态。

第一版规格：

- 默认支持 `50` 个手动快照。
- 手动快照永久保存，除非用户主动删除。
- 快照可以命名。
- 快照显示缩略图。
- 点击快照可完整回到那个瞬间。
- 回退快照前，自动生成一个“回退前快照”，防止误操作。

### 快照复制图层

第一版可以预留，若开发量允许则加入。

功能定义：

- 用户可以浏览某个快照中的图层列表。
- 可以从快照复制单个图层到当前状态。
- 可以从快照复制多个图层到当前状态。
- 当前文件不会整体回退。
- 复制操作进入普通撤销历史。

如果第一版时间紧张，此功能可推迟到中间优化版本，但底层快照结构要支持。

### 自动保存

自动保存和快照分开。

自动保存只用于：

- 崩溃恢复。
- 意外退出恢复。
- 未保存文件恢复。

自动保存不显示在快照列表里，避免混乱。

## 智能辅助

第一版只做轻量辅助，不做高级透视和人偶。

### QuickShape

触发方式：

- 用户画完一笔后笔尖停驻不松开。
- 达到短暂停驻阈值后自动修正。
- 不弹提示面板。

支持形状：

- 直线。
- 圆/椭圆。
- 正圆。
- 矩形/正方形。
- 正三角形。
- 正五边形、正六边形可选。

要求：

- 修正后保留当前笔刷质感。
- 沿修正路径重新铺当前笔刷。
- 支持 Shift 吸附水平、垂直、15 度角、正圆、正方形。
- 可在设置中关闭。

### 智能闭合

用于填色、选区、魔棒。

要求：

- 小缝隙可以被识别为闭合区域。
- 不一定真实补线。
- 可设置闭合容差。

默认界面必须干净。

常驻显示：

- 画布。
- 当前工具。
- 当前颜色。
- 笔刷大小。
- 图层入口。
- 撤销/重做。
- 保存状态。

可折叠面板：

- 图层。
- 颜色。
- 笔刷库。

临时浮层：

- 选区边界。
- 变换框。
- 笔刷大小圆圈。

不做：

## 第一版验收标准

- 在用户当前 MacBook Air + Wacom 影拓 5 上可以稳定读取压感并绘制。
- 可以新建 `6000 x 6000` 画布并流畅绘制。
- 可以创建多图层作品。
- 压感大小和浓度自然。
- 稳定器具备接近 SAI 的顺滑感。
- 快照可以完整回退。
- 自动保存可以恢复异常退出前的作品。
- 可以导出 PNG/JPEG。
- 默认界面干净，用户可以快速进入绘画状态。
